# 备份管理 API 文档

## 📋 模块概述

备份管理模块提供系统数据备份的创建、查询和删除功能。

**接口数量**: 3个
**基础路径**: `/api/v1/backup`
**认证要求**: 所有接口均需要认证 ✅
**权限级别**: 仅管理员

---

## 📑 主要接口列表

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建备份 | POST | `/` | 触发手动备份 |
| 获取备份列表 | GET | `/` | 获取备份文件列表 |
| 删除备份 | DELETE | `/:filename` | 删除备份文件 |

---

## 核心接口说明

### 10.1 创建备份

**端点**: `POST /api/v1/backup/`
**认证**: ✅ 需要认证（管理员）
**功能**: 手动触发系统数据备份

**成功响应**:
```json
{
  "message": "备份创建成功",
  "data": {
    "filename": "backup_20251228_120000.sql",
    "size": 1024000,
    "created_at": "2025-12-28T12:00:00.000Z"
  }
}
```

---

### 10.2 获取备份列表

**端点**: `GET /api/v1/backup/`
**认证**: ✅ 需要认证（管理员）

**响应**:
```json
{
  "message": "获取成功",
  "data": [
    {
      "filename": "backup_20251228_120000.sql",
      "size": 1024000,
      "created_at": "2025-12-28T12:00:00.000Z"
    }
  ]
}
```

---

## 备份说明

- **备份内容**: 数据库完整备份
- **存储位置**: `backend/backups/` 目录
- **文件命名**: `backup_YYYYMMDD_HHMMSS.sql`
- **保留策略**: 建议保留最近7天的备份
